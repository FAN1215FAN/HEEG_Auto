# V2断言资产规范

## 目标

断言资产用于把页面结果判断集中治理，case 中只写断言名称，不直接写底层判断细节。

## 推荐字段

```yaml
断言资产:
  - 断言标识:
    中文名称:
    用途说明:
    检查项:
      - 名称:
        动作:
        窗口:
        元素:
        文本:
        超时:
        可选:
```

## 推荐写法

```yaml
断言: 创建患者成功
```

不推荐把底层判断直接展开到 case 里。

## 当前支持的检查动作

- 等待窗口
- 断言存在
- 断言窗口关闭
- 断言文本可见

## 当前能力补充

- 断言检查项支持 `${变量}` 占位符。
- 可引用顶层 `变参`。
- 也可引用前序步骤中已填写过的字段值，例如 `${患者姓名}`。

## 当前项目常用断言

- 患者主界面已出现
- 创建患者窗口打开
- 创建患者页面关闭
- 创建患者成功
- 设备设置窗口已出现
- 设备设置已关闭
