# 断言目录

本文用于说明当前正式模块已定义的断言组，以及每个断言组对应的判定要求和方法。机器可维护源文件位于：

- `src/heeg_auto/config/assertion_groups.yaml`

## 新建患者

### 创建成功

- 判定对象：新建患者流程
- 判定条件：点击确定后创建患者窗口关闭，且主界面可见当前患者姓名
- 成功标准：两条断言均通过
- 失败标准：任一断言失败或窗口未正常关闭
- 依据来源：UI界面标题与主界面文本
- 备注：对应 `patient.create` 模块的正向创建场景

### 创建失败

- 判定对象：新建患者流程
- 判定条件：创建患者窗口仍存在，且可见预期错误文本（若配置）
- 成功标准：窗口保留且错误提示符合预期
- 失败标准：窗口被关闭或错误提示不符合预期
- 依据来源：UI界面标题与错误提示文本
- 备注：对应 `patient.create` 模块的异常输入场景

## 设备设置

### 设置成功

- 判定对象：设备设置流程
- 判定条件：点击确定后设备设置窗口关闭
- 成功标准：设备设置窗口从当前窗口树中消失
- 失败标准：设备设置窗口仍存在或关闭状态无法确认
- 依据来源：UI界面标题
- 备注：对应 `device.settings` 模块的正向设置场景

### 设置失败

- 判定对象：设备设置流程
- 判定条件：点击提交后设备设置窗口仍存在
- 成功标准：设备设置窗口继续存在，便于后续补充错误提示断言
- 失败标准：设备设置窗口意外关闭
- 依据来源：UI界面标题
- 备注：当前先保留结构，后续按业务需要补充具体错误提示断言

## 启动软件

### 当前说明

- `system.launch` 当前不要求显式断言组。
- 该模块主要负责建立或复用应用会话，后续如需补充启动成功判定，可在此文档与 `assertion_groups.yaml` 中统一登记。
