# 维护流程

## 当前维护原则

- 新增 formal case 时，默认使用 V2 步骤式结构。
- 不再继续扩写新的 V1 模块链业务 case。
- 新增 case 前先补窗口资产、元素资产、断言资产。
- 改动作能力时必须同步：
  - `base_page`
  - `core.actions`
  - `actions.registry`
  - `v2.executor`
  - 测试
  - 文档

## 当前推荐顺序

1. 先补或确认自动化标签。
2. 再补 V2 资产。
3. 再写 case。
4. 再补测试。
5. 最后更新文档和 changelog。

## 当前验证要求

- 至少跑相关定向测试。
- 交付前跑 `python -m pytest -q`。
- 如果运行语义变了，必须同步更新 `README.md` 和 `docs/运行指南.md`。
