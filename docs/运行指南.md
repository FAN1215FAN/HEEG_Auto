# 运行指南

## 环境准备

1. 在项目根目录创建虚拟环境：`python -m venv .venv`
2. 激活虚拟环境
3. 安装依赖：`pip install -r requirements.txt`
4. 推荐在 VS Code 中打开项目目录运行

## 推荐运行方式

- 单次演示：`python run_demo.py`
- 多 case 选择运行：`python run_case_suite.py`（生成 `artifacts/reports/HEEG_Auto_Report_时间戳.html`）
- 结构与治理层测试：`python -m pytest`
- 真实 UI 套件：`python -m pytest -m ui --run-ui`（治理/联调使用）
- 导出控件树：`python run_inspector.py`

## 当前默认运行内容

`run_demo.py` 当前默认执行：
- `src/heeg_auto/cases/患者检查管理/患者管理/1新建患者_正常创建.yaml`

## 正式用例结构

当前正式用例采用“模块调用式”结构，推荐直接写中文字段：

```yaml
用例编号: TC_LONG_001
用例名称: 设备设置_采样率校验
标签:
  - smoke
  - device
  - patient

变参:
  模块: 设备设置
  参数: 采样率
  候选值:
    - 1000
    - 2000
    - 4000

循环次数: 2
失败即停: 是

模块链:
  - 模块: 启动软件
    参数:
      会话模式: 自动

  - 模块: 设备设置
    参数:
      设备类型: Neusen HEEG
      采样率: ${变参值}
      设备增益: 6
      IP地址: 192.168.1.100
    断言组: 设置成功

  - 模块: 新建患者
    参数:
      患者姓名: 张三
      性别: 女
      利手: 左手
      病历号: 123
      脑电号: 456
      备注: 多模块样板
    断言组: 创建成功
```

## 单个 case 多次循环

如果只想让单个 case 重复执行多次，直接在该 case 顶层增加 `循环次数` 即可，例如：

```yaml
用例编号: TC_PATIENT_001
用例名称: 新建患者_正常创建
循环次数: 3
失败即停: 是

模块链:
  - 模块: 新建患者
    参数:
      患者姓名: 哈哈123
      性别: 女
      利手: 右手
      病历号: test123
      脑电号: 123456
      备注: 正向模块化样板
    断言组: 创建成功
```

然后运行：

1. 执行 `python run_case_suite.py`
2. 输入该 case 的序号、用例编号，或只选它所在目录
3. 框架会按 `循环次数` 连续执行

执行规则：
- `循环次数: 3` 就表示整条 case 连续跑 3 轮
- 如果该 case 不写 `启动软件` 模块，就表示默认在软件已经启动的前提下直接复用当前会话执行；执行前会自动连接已有应用
- 如果同时配置了 `变参`，执行顺序为“先变参值，再循环轮次”
- 只要任一轮失败，后续轮次统一标记为 `NOT_RUN`
- 日常 `run_case_suite.py` 执行不会额外追加非 case 指定的关软件/开软件动作；只有步骤卡住 60 秒没有新进展时，watchdog 才会强制结束软件

## 多轮执行规则

- `变参` 会把同一 case 按候选值展开为多轮独立执行
- `循环次数` 会把同一 case 再按轮次重复执行
- 两者可组合，执行顺序为“先变参值，再循环轮次”
- 只要任一轮失败，后续轮次统一标记为 `NOT_RUN`
- 报告会展示每一轮的执行名称、执行参数、循环轮次、最终状态和失败原因/断言信息

## 参数书写规则

### 默认规则

- 默认所有参数不加双引号
- YAML 中出现的数字、IP、编号等值，加载后会统一按文本处理
- `断言组` 必须写业务名称，不再写 `预期状态: PASS/FAIL`

### 值中包含双引号时

如果软件里需要实际输入双引号字符，请用单引号包裹整段文本，例如：

```yaml
病历号: '"123"'
备注: '患者姓名包含"双引号"'
```

## 启动软件配置

启动软件路径不再写在 case 中，而是统一维护在：

- `src/heeg_auto/config/app_config.yaml`

当前 `启动软件` 模块只需要配置：

- `会话模式`

支持的会话模式：

- `自动`：允许启动新应用，若已有可复用会话则直接复用/连接
- `复用已有应用`：要求复用当前已打开应用，如果未检测到则直接报错

## 断言组说明

当前 case 中每个模块都应显式写 `断言组`，例如：

- `设置成功`
- `设置失败`
- `创建成功`
- `创建失败`

断言组的判定标准、依据和备注见：

- `docs/断言目录.md`
- `src/heeg_auto/config/assertion_groups.yaml`

## 目录级初始化/清理

当前已移除目录级初始化/清理功能，真实 UI 套件不再在进入或离开某个 case 目录时自动执行目录级 before/after。

原因是按目录批量运行时，这种统一前后置容易与具体 case 的会话策略冲突，导致误关软件或打断复用场景。后续如果要重新设计环境初始化/清理，会基于更明确的规则单独实现。

## run_case_suite.py

可以在 VS Code 中直接运行 `run_case_suite.py`。
运行后会先列出所有正式 case 和可批量运行的目录，例如：

- `设备设置_02 | 设备设置_默认参数 | 目录: 系统设置/设备设置 | 模块链: 设备设置 | 循环: 5次`
- `设备设置_01 | 设备设置_采样率校验 | 目录: 系统设置/设备设置 | 模块链: 启动软件 -> 设备设置 | 变参: 设备设置 / 采样率 / 3值`
- `患者管理 (2条)`

支持输入：

- `all` 或 `全部`：运行全部 case
- `1,2`：按序号选择多个 case
- `患者管理_01,设备设置_01`：按用例编号选择多个 case
- `患者管理` 或 `设备设置`：按文件夹名称批量运行该文件夹下所有 case

执行顺序以你的输入顺序为准。`run_case_suite.py` 当前只支持输入文件夹名称本身，不再要求 `目录:` 前缀，也不再接受完整目录路径。

运行完成后，`run_case_suite.py` 只会输出一份本次执行报告：

- `artifacts/reports/HEEG_Auto_Report_时间戳.html`

日常查看和汇报直接打开这份 HTML 报告即可。`HEEG_Auto_Report_时间戳.html` 当前会保留执行概览、开始时间、结束时间、执行结果摘要、模块参数快照、异常/未执行轮次、失败原因/断言信息和截图产物链接；不会再展开记录成功步骤明细。

## pytest 用途

pytest 当前负责治理层，而不是日常正式 case 执行层。它主要承担：
- 收集正式 case，并在测试名中展示 `用例编号 | 用例名称 | 目录 | 模块链`
- 收集注册的大模块，并展示 `模块标识 | 模块名称`
- 校验 case、元素清单、模块定义、断言组和报告结构
- 驱动可选的真实 UI smoke 联调
- 输出 pytest HTML 治理报告
