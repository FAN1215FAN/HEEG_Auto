# 运行指南

## 日常入口

- `python run_demo.py`
  - 走日常演示入口
- `python run_case.py`
  - 走人工选择入口
- `python run_inspector.py`
  - 导出控件树

## pytest 入口

- `python -m pytest`
  - 跑治理测试和单元测试
- `python -m pytest --run-formal --run-ui -s`
  - 执行 formal UI case
- `python -m pytest --case-dir 设备设置 -s`
  - 只执行某个业务目录
- `python -m pytest --case-id 患者管理_01,设备设置_V2_01 -s`
  - 按 case 编号执行
- `python -m pytest --case-file src/heeg_auto/cases/系统设置/设备设置/设备设置_V2_采样率校验.yaml -s`
  - 按单个 case 文件执行

## 当前执行约束

- 业务 case 不再在本体内启动软件。
- 目录 `init.yaml` 负责启动和恢复主界面。
- case 只写业务动作和断言。
- formal case 当前全部按 V2 结构执行。

## 推荐调试顺序

1. 先跑 `python -m pytest tests/test_v2_case_loader.py tests/test_v2_executor.py -q`
2. 再跑 `python -m pytest tests/test_case_loader.py tests/test_case_catalog.py -q`
3. 最后跑 `python -m pytest -q`

## 失败排查顺序

1. 先看 case YAML 结构是否符合 V2 规范。
2. 再看窗口资产、元素资产、断言资产命名是否一致。
3. 再看 AutomationId 是否缺失或控件类型是否配错。
4. 最后再查动作层和 pywinauto 定位问题。
