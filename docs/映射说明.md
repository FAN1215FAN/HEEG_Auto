# 映射关系说明

本文用于说明“自然语言 -> 内部标识 -> 代码实现”的固定映射关系。这部分属于长期资产，后续新增动作、元素、业务模块或断言组时必须同步维护。

## 1. 动作映射

动作注册位置：`src/heeg_auto/actions/registry.py`

动作执行实现：`src/heeg_auto/core/actions.py`

当前主线的典型动作包括：

- `启动应用` -> `launch_app`
- `单击` -> `click`
- `输入` -> `input_text`
- `下拉选择` -> `select_combo`
- `选择单选` -> `select_radio`
- `等待窗口` -> `wait_for_window`
- `断言存在` -> `assert_exists`
- `断言窗口关闭` -> `assert_window_closed`
- `断言文本可见` -> `assert_text_visible`

## 2. 元素映射

元素目录：`src/heeg_auto/elements/`

元素解析：`src/heeg_auto/elements/loader.py`

元素层负责维护稳定控件资产，例如：

- 业务中文标签
- 页面归属 `page`
- `automation_id`
- `control_type`
- `title`
- 兼容别名 `aliases`

## 3. 模块映射

模块注册位置：`src/heeg_auto/modules/registry.py`

模块定义目录：`src/heeg_auto/modules/`

模块加载位置：`src/heeg_auto/modules/loader.py`

模块执行位置：`src/heeg_auto/runner/模块执行器.py`

当前主线模块包括：

- `启动软件` -> `system.launch`
- `新建患者` -> `patient.create`
- `设备设置` -> `device.settings`

模块层的作用不是存零散动作，而是定义完整业务过程。模块 YAML 会引用元素清单，并用中文动作描述步骤和命名断言组。

## 4. case 映射

case 目录：`src/heeg_auto/cases/`

case 加载位置：`src/heeg_auto/runner/case_loader.py`

pytest 展示支撑：`tests/support/case_catalog.py`

正式 case 当前使用中文 YAML，并以 `模块链` 为核心结构。当前 case loader 会把以下中文字段映射为内部稳定字段：

- `患者姓名` -> `patient_name`
- `性别` -> `gender`
- `利手` -> `habit_hand`
- `病历号` -> `patient_id`
- `脑电号` -> `eeg_id`
- `预期错误包含` -> `expect_error_contains`
- `设备类型` -> `device_type`
- `采样率` -> `sample_rate`
- `会话模式` -> `session_mode`
- `变参值` -> `variant_value`

同时它还负责：

- 规范化裸值参数为文本
- 解析 `${timestamp}` 与 `${变参值}`
- 把中文模块名映射为稳定模块 id
- 读取 `变参`、`循环次数`、`失败即停`

## 5. 断言组映射

断言目录位置：`src/heeg_auto/config/assertion_groups.yaml`

说明文档位置：`docs/断言目录.md`

当前 case 不再写 `预期状态: PASS/FAIL`，而是对每个模块显式指定 `断言组`。例如：

- `创建成功`
- `创建失败`
- `设置成功`
- `设置失败`

`ModuleRunner` 会读取模块 YAML 中同名断言组，并执行该组下的断言步骤。

## 6. 从自然语言到执行的转化逻辑

当前项目的关键链路可以概括为：

1. 用户在正式 case YAML 中写中文业务场景、变参和断言组
2. `FormalCaseLoader` 标准化 case 字段、参数和变量
3. `FormalCaseRunner` 按变参候选值和循环次数构建执行计划
4. `ModuleRunner` 读取模块 YAML 并执行步骤与指定断言组
5. `ElementStore` 把模块里的元素引用解析为真实 locator
6. `ActionExecutor` 通过动作 id 调用底层 UI 操作
7. `UIADriver` 和页面对象负责实际驱动 WPF 界面
8. `reporting.py` 汇总每轮结果并输出 `PASS / FAIL / NOT_RUN`

## 7. 维护要求

- 新增动作时，同时更新动作注册与底层实现。
- 新增元素时，同时更新元素清单与 `docs/控件清单.md`。
- 新增模块时，同时更新模块注册、断言目录、正式 case、pytest 治理测试和本文档。
- 中文展示名可以继续使用中文，但内部模块 id、元素 key、文件名应保持稳定可维护。
