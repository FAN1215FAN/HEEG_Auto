# 运行指南

## 文档定位

这份文档说明当前项目怎么运行、各入口分别给谁用、执行时有哪些约束，以及推荐的验证顺序。

## 一、日常入口

### 1. `python run_case.py`

用途：

- 给测试同事日常手工选择 case 或目录运行
- 显示可运行 case 列表和目录列表
- 共用正式 case 真源

### 2. `python run_demo.py`

用途：

- 演示入口
- 不替代正式回归入口

### 3. `python run_inspector.py`

用途：

- 导出控件树
- 辅助补窗口和元素资产

### 4. `python run_ratio_picker.py`

用途：

- 读取当前最上层窗口点击点的比例坐标
- 供图形交互类 case 手工录入比例参数
- 不参与 case 正式执行

## 二、pytest 入口

### 1. 全量治理测试

```bash
python -m pytest
```

用途：

- 跑治理测试、单元测试和当前已纳入的 smoke 测试

### 2. 正式 UI case

```bash
python -m pytest --run-formal --run-ui -s
```

用途：

- 跑正式 UI case

### 3. 按环境模式执行

```bash
python -m pytest --run-formal --run-ui --environment-mode reuse_per_suite -s
python -m pytest --run-formal --run-ui --environment-mode reset_per_directory -s
```

### 4. 按目录、编号、文件筛选

```bash
python -m pytest --case-dir 设备设置 -s
python -m pytest --case-id 患者管理_01 -s
python -m pytest --case-file src/heeg_auto/cases/系统设置/设备设置/设备设置_采样率校验.yaml -s
```

### 5. Jenkins 串联执行指定 case

当该项目作为安装链路后的第二阶段自动化测试接入 Jenkins 时，当前已验证可直接在 `D:\AI_Auto\HEEG_Auto` 下执行：

```bash
python -m pytest --case-id 历史回放剪辑_01 -s
```

当前 `--case-id` 已可触发正式 case 执行范围；Jenkins 不需要再额外补一个“启动软件”命令，软件启动由目录初始化和正式执行链负责。

## 三、执行前提

1. 正式业务 case 默认不在 case 本体里负责启动软件。
2. 目录级 `init.yaml` 负责目录初始化和基础环境恢复。
3. 套件级 `suite_setup.yaml / suite_cleanup.yaml` 是可选支持文件。
4. 正式业务 case 只写业务动作、业务参数和命名断言。

## 四、环境模式说明

### 1. `reuse_per_suite`

- 当前默认推荐模式
- 套件内尽量复用同一应用会话
- 更适合串联回归

### 2. `reset_per_directory`

- 严格隔离模式
- 切换目录时重建环境
- 更适合隔离要求高但接受运行慢的场景

## 五、图形交互 case 运行说明

对于历史回放、实时采集等图形交互类 case：

1. 比例参数先通过 `run_ratio_picker.py` 辅助读取。
2. 历史回放剪辑类 case 当前只推荐在 `数据` 段填写 `剪辑开始时间 / 剪辑结束时间` 与比例参数，不再要求手填中间断言和主窗口最终断言的时长字段。
3. 正式运行时仍走 `run_case.py` 或 `pytest`。

当前不支持：

- 运行中自动取比例
- 自动回填 YAML
- 录完点后直接生成完整 case

## 六、推荐验证顺序

1. 先跑基础治理测试：

```bash
python -m pytest tests/test_asset_store.py tests/test_step_case_loader.py tests/test_step_case_executor.py -q
```

2. 再跑目录和目录生命周期相关测试：

```bash
python -m pytest tests/test_case_loader.py tests/test_case_catalog.py tests/test_directory_lifecycle.py -q
```

3. 最后跑全量：

```bash
python -m pytest -q
```
