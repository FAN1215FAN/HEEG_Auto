# 目录说明

## 文档定位

这份文档说明当前正式目录的职责。只描述当前仍有效的目录和文件，不再保留旧模块时代目录说明。

## 一、根目录

### `README.md`

- 项目总入口

### `CHANGELOG.md`

- 记录里程碑级变更和文档恢复记录

### `run_case.py`

- 测试日常执行入口

### `run_demo.py`

- 演示入口

### `run_inspector.py`

- 控件树检查入口

### `run_ratio_picker.py`

- 图形交互比例读取辅助入口

## 二、核心源码目录

### `src/heeg_auto/config`

- 项目配置、路径、常量和环境模式默认值

### `src/heeg_auto/core`

- 底层驱动
- 动作执行
- 基础页面能力
- 日志与报告
- 图形交互辅助能力

### `src/heeg_auto/pages`

- 复杂页面或窗口专用逻辑

### `src/heeg_auto/runner`

- case 加载
- case 执行
- case 调度
- 目录生命周期
- payload 字段别名处理

### `src/heeg_auto/assets`

- 正式资产总入口
- 下分：
  - `windows/`
  - `elements/`
  - `assertions/`

### `src/heeg_auto/cases`

- 正式步骤式 case
- 当前按软件功能域分组，这些顶层文件夹表示当前软件设想的大分类区，不是封闭列表
- 现行顶层包括：
  - `ERP回放/`
  - `公共功能/`
  - `启动/`
  - `回放数据/`
  - `实时录制/`
  - `患者检查管理/`
  - `系统设置/`
- 后续允许继续增加新的同级分类文件夹
- 每个分类下允许继续拆分下级文件夹
- 下级文件夹当前不强制定统一拆分原则，允许测试或实施侧按实际维护需要先新增
- 后续是否按手工用例分类、按窗口/功能分层，或按其他方式统一，等真实使用一段时间后再评审
- `历史回放剪辑` 当前归属 `回放数据/历史回放剪辑/`
- 目录下允许存在：
  - `init.yaml`
  - 正式业务 case
- 根目录可选存在：
  - `suite_setup.yaml`
  - `suite_cleanup.yaml`

## 三、测试目录

### `tests`

- 治理测试
- smoke 测试
- pytest 桥接能力

### `tests/support`

- case 目录扫描
- pytest 展示和筛选辅助

## 四、产物目录

### `artifacts`

- 日志
- 截图
- HTML 报告
- 检查工具输出

## 五、文档目录

### `docs`

当前包含四类内容：

1. 主文档
   - 项目书
   - 框架收敛方案
   - 瘦身迁移清单
   - 需求说明
2. 资产与结构文档
   - 资产总表
   - 资产缺口清单
   - 架构说明
   - 目录说明
   - 运行指南
3. 专题文档
   - 历史回放剪辑等专题规则已经并回主文档和资产规范文档
4. 模板
   - 步骤式 case 模板
   - suite 支持文件模板

## 六、当前已移除的旧面

以下内容已经不再作为正式结构存在：

1. 旧模块目录
2. 旧元素目录
3. 旧兼容目录
4. `V1 / V2` 双轨对外正式命名
5. docs 中旧模块时代示例骨架
