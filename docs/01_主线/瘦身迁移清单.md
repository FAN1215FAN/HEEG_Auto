# 瘦身迁移清单

## 文档定位

这份文档用于跟踪代码与目录层面的瘦身迁移，不是结果摘要。每完成一个阶段，都必须在这里留下已完成事项、未完成事项和风险说明。

## 总体目标

把项目从历史双轨和多套过渡结构，收敛成一套正式可维护的步骤式自动化工程结构，同时不破坏测试入口、研发入口和现有资产沉淀。

## 迁移原则

1. 先建立正式入口，再切旧入口，不做硬切断后再补救。
2. 旧目录、旧示例、旧模板必须物理移除或归档，不能继续混在正式面。
3. 运行时代码与文档必须同步瘦身，不能只动代码不动文档。
4. 环境策略是独立议题，不能在瘦身过程中顺手改乱。
