# V2执行器设计说明

## 设计目标

V2 的目标不是给 V1 模块链再套一层壳，而是让 case 编写彻底去模块化。
当前统一采用：`步骤 / 窗口 / 参数 / 元素 / 按钮 / 动作 / 断言`。

## 顶层结构

```yaml
用例编号:
用例名称:
标签:
变参:
循环次数:
失败即停:
步骤:
```

## 步骤结构

```yaml
- 名称:
  窗口:
  参数:
  元素:
  按钮:
  动作:
  断言:
  超时:
  可选:
```

## 默认规则

### 动作默认推断

- 写了 `按钮`：默认点击
- 写了 `元素` 且没写 `动作`
  - `Button` -> 点击
  - `RadioButton` -> 选择单选
  - `Edit / Custom / Text` + `值` -> 输入
  - `ComboBox` + `值` -> 下拉选择
  - `CheckBox` + `值` -> 设置勾选
- 写了 `参数`
  - `Edit` -> 输入
  - `ComboBox` -> 下拉选择
  - `RadioButton` -> 选择单选
  - `CheckBox` -> 设置勾选
- 只有 `窗口`
  - 默认等待窗口出现

### 窗口作用域

- 同一步内的 `参数 / 元素 / 按钮` 都默认在该 `窗口` 下解析。
- 不做跨步骤窗口自动猜测。

### 断言执行

- `断言` 仍然走命名断言资产。
- 断言检查项内部可引用前序步骤写入的值，例如 `${患者姓名}`。

## CheckBox 是什么

这里的 `CheckBox` 指复选框控件。
在 V2 里它对应“勾选 / 取消勾选”这类操作，推荐写法是放在 `参数:` 下，例如：

```yaml
- 名称: 设置通用选项
  窗口: 通用设置
  参数:
    自动上传: 是
```

执行器会根据资产里该元素的 `ControlType: CheckBox` 自动推断为 `设置勾选`。
