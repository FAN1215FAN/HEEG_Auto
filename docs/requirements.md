# 需求说明

## 项目目标

创建一个基于 Python 的 WPF 桌面 UI 自动化标准工程，验证 `Neuracle.EEGRecorder.Viewer.HEEG.exe` 在“新增患者”场景下的自动化可行性，并作为后续可复用测试自动化的基础。

## 当前范围

- 启动指定 exe
- 在主界面点击“新增”按钮
- 打开“创建患者”弹窗
- 输入患者基础信息并点击“确定”
- 在主界面患者列表中验证新增患者姓名已出现
- 输出日志、截图、pytest HTML 报告
- 验证中文行式脚本是否能作为更低门槛的自动化表现形式
- 每次运行自动生成一份全中文 JSON 报告和 Word 报告

## 当前不包含

- 可视化拖拽平台
- 通用业务流程编排器
- 复杂日期控件操作
- 多业务场景覆盖
- 测试数据清理策略
- 成功截图批量入报告

## 工程要求

- 编码语言固定为 Python
- 基于 `pywinauto` 的 UIA 自动化实现
- 同时保留结构化 YAML 用例和中文行式脚本原型
- 采用标准化目录和文档约束
- 保留未来扩展更多通用动作和更多业务窗口的能力
- 运行产物至少包含日志、失败截图、pytest HTML 报告、JSON 报告和 Word 报告
