# 架构说明

## 当前架构结论

当前项目已经形成两层结构：
- 执行层：`driver / pages / core.actions / runner`
- 资产层：`cases / modules / v2/assets`

其中 formal 执行主线已经切换为 V2 步骤式 case，V1 模块链只保留兼容能力，不再继续扩写新的业务 case。

## 执行链路

### 目录级支持文件

- `init.yaml`：负责启动软件、恢复主界面、建立可执行环境。
- `cleanup.yaml`：目录支持文件，当前保留空清理语义。

### formal case

formal case 统一通过 [formal_case_runner.py](/D:/AI_project/AI_Auto/HEEG_Auto/src/heeg_auto/runner/formal_case_runner.py) 调度：
- 识别 V1 / V2 case 格式
- 进入目录生命周期管理
- 执行 case
- 汇总报告与失败产物

### V2 执行器

V2 的核心链路如下：
- [case_loader.py](/D:/AI_project/AI_Auto/HEEG_Auto/src/heeg_auto/v2/case_loader.py)：解析中文步骤式 YAML
- [asset_store.py](/D:/AI_project/AI_Auto/HEEG_Auto/src/heeg_auto/v2/asset_store.py)：加载窗口、元素、断言资产
- [executor.py](/D:/AI_project/AI_Auto/HEEG_Auto/src/heeg_auto/v2/executor.py)：执行步骤、推断默认动作、执行命名断言

## 当前 V2 规则

- 业务 case 不再写启动软件步骤。
- `窗口` 是步骤作用域。
- `按钮` 默认推断为点击。
- `参数` 按元素控件类型自动推断动作为输入、下拉选择、单选或勾选。
- `断言` 继续命名化，不在 case 里写底层判断细节。
- 断言现在支持读取前序步骤里填写过的值，例如 `${患者姓名}`。

- 双击

- 下拉选择

## 当前资产治理

- 窗口资产：定义窗口锚点和标题语义
- 元素资产：定义窗口内的稳定控件
- 断言资产：集中维护页面结果判断
