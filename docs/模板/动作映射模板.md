# 动作映射表模板

用于固定“中文动作 -> 内部动作 ID -> 参数格式 -> 实现位置”的关系。

## 使用原则

1. 一旦动作对外发布，原则上不随意改名。
2. 中文动作和内部动作 ID 必须一一对应。
3. 参数格式必须写清楚。
4. 实现位置写真实当前文件，而不是假想拆分文件。

## 模板

| 中文动作 | 内部动作 ID | 主要参数 | 实现位置 | 备注 |
| --- | --- | --- | --- | --- |
| 点击 | `click` | `window / element / button / timeout` | `src/heeg_auto/core/actions.py` | 默认高频动作 |
| 输入 | `input_text` | `window / element / value / timeout` | `src/heeg_auto/core/actions.py` | 表单高频动作 |
| 下拉选择 | `select_combo` | `window / element / value / timeout` | `src/heeg_auto/core/actions.py` | 表单高频动作 |
| 等待窗口 | `wait_for_window` | `window / timeout` | `src/heeg_auto/core/actions.py` | 断言和等待都常用 |
| 拖动 | `drag` | `window / params / timeout` | `src/heeg_auto/core/actions.py` | 图形交互类动作 |

## 建议字段说明

- 中文动作：写在 case 或断言资产中的动作名
- 内部动作 ID：执行器调用的稳定动作标识
- 主要参数：最关键参数集合
- 实现位置：真实当前代码位置
- 备注：兼容说明、是否默认动作、是否仅图形交互使用等
