# 参数设计说明

## 文档定位

这份文档说明当前正式 case 中参数、数据、变参的职责边界和命名要求。

## 一、参数分层

当前正式步骤式 case 里，参数主要分三层：

### 1. `数据`

用途：

- 承载当前场景中频繁修改的业务变量
- 作为整条 case 的上下文输入

适合放在 `数据` 中的内容包括：

- 患者姓名
- 病历号
- 脑电号
- 备注
- 设备类型
- 采样率
- 图形交互比例
- 历史回放剪辑的开始时间、结束时间、记录时长

### 2. 步骤内 `参数`

用途：

- 在当前步骤中做表单填写
- 在显式动作步骤中传入动作参数

### 3. `变参`

用途：

- 表示同一条 case 需要按多组候选值独立执行

当前结论：

- 只有“候选值中的每一组都必须独立跑一轮 case”时，才使用 `变参`
- 图形交互类 case 当前不推荐滥用 `变参`

## 二、当前高频业务参数

### 患者相关

- 患者姓名
- 性别
- 利手
- 病历号
- 脑电号
- 备注

### 设备设置相关

- 设备类型
- 采样率
- 波特率
- 头盒数目
- IP 地址
- 端口
- 设备增益

### 环境与执行相关

- 会话模式
- 软件路径
- 失败即停
- 循环次数

### 图形交互相关

- 波形起点 X 比例
- 波形起点 Y 比例
- 波形终点 X 比例
- 波形终点 Y 比例
- 剪辑开始时间
- 剪辑结束时间
- 主窗口新增记录时长

## 三、命名原则

1. YAML 中优先使用中文参数名。
2. 运行时统一映射到稳定英文键。
3. 相同业务含义在不同 case 中尽量保持同名。
4. 图形交互类 case 的数据字段命名尽量统一，避免同义不同名。

## 四、历史回放剪辑专题结论

对于历史回放剪辑类 case：

1. 频繁修改的时间和比例字段统一放进 `数据` 段。
2. 中间窗口片段断言与主窗口最终记录断言使用不同字段，不再混用一个“时长”变量。
3. 比例参数继续人工录入，但通过比例辅助工具降低输入成本。

### 历史回放剪辑类 case 推荐参数

| 参数名 | 示例 | 说明 |
| --- | --- | --- |
| 记录名称 | `自动化测试数据` | 目标检查记录名 |
| 原始记录时长 | `00:00:42` | 主窗口原始父记录时长 |
| 剪辑开始时间 | `00:00:01` | 数据剪辑窗口中的开始时间 |
| 剪辑结束时间 | `00:00:02` | 数据剪辑窗口中的结束时间 |
| 主窗口新增记录时长 | `00:00:02` | 主窗口新增子记录时长 |
| 波形起点X比例 | `0.14` | 比例工具读取 |
| 波形起点Y比例 | `0.55` | 比例工具读取 |
| 波形终点X比例 | `0.28` | 比例工具读取 |
| 波形终点Y比例 | `0.55` | 比例工具读取 |

当前结论：

1. `剪辑开始时间 / 剪辑结束时间` 用于中间窗口片段断言。
2. `主窗口新增记录时长` 用于主窗口最终结果断言。
3. 这三类时间字段不能再混用成一个“时长”参数。
