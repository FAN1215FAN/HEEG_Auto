# 断言目录

## 文档定位

这份文档说明当前正式断言资产的组织方式、主要断言组和当前重点缺口。正式可维护源位于 `src/heeg_auto/assets/assertions/`。

## 一、当前正式断言文件

### 推荐聚合

- `recommended_assertions.yaml`

### 业务断言

- `创建患者.yaml`
- `设备设置.yaml`
- `实时采集与阻抗.yaml`
- `历史回放剪辑.yaml`

## 二、当前主要断言组

### 1. 主界面基础断言

代表断言：

- 患者主界面已出现

用途：

- 软件启动后或环境恢复后确认主界面可见

### 2. 创建患者断言

代表断言：

- 创建患者窗口打开
- 创建患者页面关闭
- 创建患者成功

用途：

- 覆盖新建患者窗口打开、关闭、提交成功等关键结果

### 3. 设备设置断言

代表断言：

- 设备设置窗口已出现
- 设备设置已关闭
- 模拟器确认窗口已出现

用途：

- 覆盖设备设置窗口打开与关闭结果

### 4. 实时采集与阻抗断言

代表断言：

- 实时采集窗口已出现
- 阻抗窗口已出现

### 5. 历史回放剪辑断言

代表断言：

- 历史回放窗口已出现
- 历史回放窗口已关闭
- 数据剪辑窗口已出现
- 剪辑完成窗口已出现
- 剪辑完成窗口已关闭
- 数据剪辑片段已生成
- 主窗口已新增最新剪辑记录

用途：

- 覆盖历史回放、数据剪辑、中间片段结果和主窗口最终结果
- 其中“数据剪辑片段已生成”和“主窗口已新增最新剪辑记录”当前统一采用时长范围断言，范围默认由 `剪辑开始时间 / 剪辑结束时间` 自动推导，默认容差为 `±1` 秒

## 三、当前断言组织原则

1. case 中优先引用命名断言，不直接展开复杂底层判断。
2. 一个命名断言内部可以包含多个检查项。
3. 业务结果优先沉淀成断言资产，不散落进 case。

## 四、当前重点缺口

1. 脑电波形和图形结果类断言模板仍不足。
2. 历史回放和数据剪辑的细粒度领域断言仍需继续补厚。
3. 测试侧断言入口还不够友好，后续需要进一步模板化。
