# 步骤执行器设计说明

## 文档定位

这份文档说明 `StepCaseExecutor` 当前负责什么、怎么推断动作、怎么处理表单步骤、断言和执行计划。

## 一、目标

把步骤式 YAML 中的步骤转成统一动作调用，并保持：

- 业务写法足够简单
- 底层动作足够稳定
- 断言和变量可复用

## 二、当前职责

1. 解析动作别名
2. 根据窗口、元素、按钮和控件类型推断动作
3. 处理普通步骤、表单填写步骤和断言步骤
4. 处理变参和循环形成的执行计划
5. 汇总步骤级执行结果

## 三、不负责的内容

1. 目录生命周期管理
2. 套件初始化策略决策
3. 资产维护流程

## 四、动作推断规则

### 1. 表单步骤

满足条件时，按表单填写处理：

- 没有显式 `动作`
- 没有 `按钮`
- 没有 `元素`
- 只有 `窗口 + 参数`

### 2. 主动作步骤

满足以下任一条件时，进入主动作执行：

- 显式写了 `动作`
- 写了 `按钮`
- 写了 `元素`
- 写了 `值` 或 `文本`

### 3. 步骤断言

- 每个步骤执行后可串接命名断言
- 断言可以读取当前上下文

## 五、执行计划

执行器会根据以下信息生成计划：

1. `循环次数`
2. `变参`
3. `失败即停`

## 六、当前支持的主要动作

- 启动应用
- 点击
- 双击
- 右键
- 拖动
- 输入
- 下拉选择
- 选择单选
- 设置复选框
- 等待窗口
- 等待可见
- 断言存在
- 断言窗口关闭
- 断言文本可见
- 断言文本不可见
- 断言最新剪辑记录
- 截图
