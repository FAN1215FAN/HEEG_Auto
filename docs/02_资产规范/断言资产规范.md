# 断言资产规范

## 文档定位

断言资产用于把业务结果判断集中治理，case 中优先只写断言名称。

## 一、正式位置

- `src/heeg_auto/assets/assertions/`

## 二、资产职责

1. 沉淀命名断言
2. 降低 case 书写成本
3. 收敛复杂业务判断

## 三、核心规则

1. 一个断言资产表达一个清晰业务结果。
2. 一个断言内部可以包含多个底层检查项。
3. case 中优先引用命名断言，不直接展开复杂检查。
4. 断言支持读取上下文变量和前序步骤写入值。

## 四、推荐字段

```yaml
断言资产:
  - 断言标识:
    中文名称:
    用途说明:
    检查项:
      - 名称:
        动作:
        窗口:
        元素:
        文本:
        超时:
        参数:
        可选:
```

## 五、当前重点方向

1. 基础断言继续由执行层直接支持。
2. 高频业务结果继续沉淀为命名断言。
3. 图形和波形结果后续继续朝模板化断言扩展。
