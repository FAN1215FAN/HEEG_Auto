# 窗口资产规范

## 文档定位

窗口资产用于定义步骤中的窗口上下文，并承接图形交互所需的窗口级标定。

## 一、正式位置

- `src/heeg_auto/assets/windows/`

## 二、资产职责

1. 提供稳定窗口名称
2. 提供窗口定位信息
3. 为元素资产和断言资产提供上下文
4. 为图形交互提供 `交互标定`

## 三、推荐字段

```yaml
窗口资产:
  - 窗口标识:
    资产类型: 窗口
    中文名称:
    所属窗口:
    AutomationId:
    ControlType:
    Name:
    ClassName:
    是否唯一:
    锚点元素:
    用途说明:
    交互标定:
```

## 四、规则

1. 一个业务窗口只保留一个清晰稳定的中文名称。
2. 业务 case 中统一引用中文名称。
3. 优先使用稳定 `AutomationId`。
4. 图形交互窗口的比例参数统一维护在窗口资产中，不散落到别处。

## 五、当前已知图形交互窗口

1. 历史回放窗口
2. 实时采集窗口

这类窗口的比例交互依赖 `交互标定`，后续如继续扩展，也应沿用同一方式。

### 历史回放剪辑专项窗口要求

当前历史回放剪辑链路至少依赖以下窗口资产：

1. 数字脑电采集记录软件窗口
2. 历史回放窗口
3. 数据剪辑窗口
4. 剪辑完成窗口

其中：

- 历史回放窗口负责维护波形区与进度条的 `交互标定`
- 数据剪辑窗口负责承接剪辑动作和片段断言
- 剪辑完成窗口负责承接确认按钮和关闭断言
