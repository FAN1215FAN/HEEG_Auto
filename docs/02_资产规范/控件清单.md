## 四、治理要求

1. 新增控件前，先确认是否已有稳定 `AutomationId`。
2. 没有稳定 `AutomationId` 时，再考虑 `Name + ControlType + ClassName` 等组合定位。
3. 自定义控件优先推动研发补自动化暴露能力，再入正式资产。
4. 同一业务含义的控件应保持统一中文命名。

## 五、当前重点缺口

1. 复杂表格和树型控件资产还不够厚。
2. 图形和波形区域更多细粒度控件仍需后续继续补。
3. 部分标题栏关闭类控件仍缺稳定标识。
