# 元素资产规范

## 文档定位

元素资产用于沉淀稳定控件定位，供步骤动作直接引用。

## 一、正式位置

- `src/heeg_auto/assets/elements/`

## 二、资产职责

1. 提供控件中文名称
2. 提供所属窗口
3. 提供定位信息
4. 为默认动作推断提供控件类型基础

## 三、核心规则

1. 元素名称优先使用测试同事容易理解的中文名称。
2. 每个元素必须绑定所属窗口。
3. 优先使用稳定 `AutomationId`。
4. 无稳定 `AutomationId` 时，再补 `Name / ControlType / ClassName` 等信息。
5. 同名元素如果跨多个窗口重复，必须通过窗口区分。

## 四、推荐字段

```yaml
元素资产:
  - 元素标识:
    资产类型: 元素
    中文名称:
    所属窗口:
    AutomationId:
    ControlType:
    Name:
    ClassName:
    是否唯一:
    锚点元素:
    用途说明:
```

## 五、当前重点注意事项

1. 标题栏关闭按钮这类高频控件要特别关注稳定标识。
2. 列表第一行、记录行这类动态行控件要优先补锚点或稳定标识。
3. 图形和波形区域更多依赖窗口交互标定，不建议把区域碎片化成大量伪元素文件。
