# 映射说明

## 文档定位

这份文档说明“中文业务表达 -> 内部稳定字段/动作 -> 实际代码执行”的固定映射关系。

## 一、动作映射

### 1. 动作实现位置

- 执行动作代码：`src/heeg_auto/core/actions.py`
- 步骤调度入口：`src/heeg_auto/runner/step_case_executor.py`

### 2. 当前主线常用动作

| 中文动作 | 内部动作 ID |
| --- | --- |
| 启动应用 | `launch_app` |
| 点击 / 单击 | `click` |
| 双击 | `double_click` |
| 右键 | `right_click` |
| 拖动 | `drag` |
| 输入 | `input_text` |
| 下拉选择 | `select_combo` |
| 选择单选 | `select_radio` |
| 设置复选框 | `set_checkbox` |

## 四、资产映射

### 1. 资产目录

- `src/heeg_auto/assets/windows/`
- `src/heeg_auto/assets/elements/`
- `src/heeg_auto/assets/assertions/`

### 2. 资产入口

- `src/heeg_auto/assets/asset_store.py`

### 3. 解析规则

- 窗口可按窗口标识或中文名称解析
- 元素优先按元素标识解析
- 若元素名称在多个窗口重复，必须显式指定窗口
- 断言按断言标识或中文名称解析

## 五、从中文 YAML 到真实执行的链路

1. 测试在步骤式 YAML 中写业务场景。
2. `StepCaseLoader` 标准化字段、数据、变参和上下文。
3. `FormalCaseRunner` 建立执行会话、看门狗和失败工件留存。
4. `StepCaseExecutor` 解析每个步骤的动作和断言。
5. `AssetStore` 解析窗口、元素、断言资产。
6. `ActionExecutor` 通过 `pywinauto + UIA` 驱动真实 WPF 界面。
7. `reporting.py` 和 pytest 汇总结果。

## 六、维护要求

1. 新增动作时，必须同步更新这里。
2. 新增参数别名时，必须同步更新这里。
3. 新增正式资产解析规则变化时，必须同步更新这里。
