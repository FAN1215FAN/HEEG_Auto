# 对话总结

## 当前项目主线

项目当前已经从早期平铺脚本方案收敛为：
- 动作层
- 元素层
- 大模块层
- 正式用例层
- pytest 执行层

并明确：
- 不再把“小模块”作为公开架构层
- 一个 case 可以串多个大模块
- 正式用例继续采用中文 YAML
- pytest 负责执行与治理，不负责直接编辑业务用例
- VS Code 为默认开发与运行入口

## 近期关键收敛

### 1. 显式启动软件模块
原先软件启动由 runner 外层通过会话策略隐式处理，现在已经改为显式系统模块：
- `启动软件` -> `system.launch`

这样 case 编辑层能直接看到真实执行结构。

### 2. 正式 case 统一写成模块链
当前 case 统一使用 `模块链`，即使只有一个业务模块也不单独发明新结构，保持后续串多个模块时的统一性。

### 3. run_case 顺序执行
`run_case.py` 现在会按用户输入的顺序执行 case。例如输入 `2,1,3`，就按 `2 -> 1 -> 3` 跑。

### 4. 设备设置大模块接入
已接入第二个大模块：
- `设备设置` -> `device.settings`

并生成了：
- 元素清单
- 大模块定义
- 正式 case
- pytest 结构测试
- 映射文档更新

### 5. 全局 skill 建立
已经建立并验证全局 skill：
- `wpf-module-importer`

后续用于根据控件 Excel 扩充新的 WPF 大模块。
